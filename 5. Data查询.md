# data 查询

- 实现了模块化后，有些outpu则需要动态查询到，比如创建`EC2`的时候，需要知道 `VPC`以及` Subnet`等信息，但他们分别在2个模块，那应该怎么处理呢？
-  如何动态（on-the-fly)查询一些值，比如创建vpc和subnet后，他们的ID是多少呢？ TF提供了data来解决这个问题

# data module的例子

```
# main.tf

data "aws_vpc" "vpc" {
  filter {
    name   = "tag:Name"
    values = [var.vpc_name]
  }
}

data "aws_subnets" "private" {

  filter {
    name   = "vpc-id"
    values = [data.aws_vpc.vpc.id]
  }
  filter {
    name   = "tag:Tier"
    values = ["Private"]
  }

}

data "aws_subnets" "public" {
  filter {
    name   = "vpc-id"
    values = [data.aws_vpc.vpc.id]
  }
  filter {
    name   = "tag:Tier"
    values = ["Public"]
  }
}

data "aws_subnet" "private" {
  for_each = toset(data.aws_subnets.private.ids)
  id       = each.value
}

data "aws_subnet" "public" {
  for_each = toset(data.aws_subnets.public.ids)
  id       = each.value
}


############################
# OUTPUT
############################

output "vpc" {
  value = data.aws_vpc.vpc
}

output "vpc_id" {
  value = data.aws_vpc.vpc.id
}

output "aws_private_subnets_ids" {
  value = data.aws_subnets.private.ids
}

output "aws_public_subnets_ids" {
  value = data.aws_subnets.public.ids
}

output "aws_private_subnets" {
  value = data.aws_subnet.private
}

output "aws_public_subnets" {
  value = data.aws_subnet.public
}
```

* 定义variables.
```
variable "environment" {
  default = "development"
}

variable "vpc_name" {}
```

# 调用data 模块
```
├── backend.tf
├── data.tf
├── locals.ew1.production.yaml
├── locals.tf
├── main.tf
└── providers.tf
```
从上面看，data模块需要 `environment` 和 `vpc_name` 这2个变量。
```
# data.tf
module "data" {
  source      = "git@github.com:mycompany/mycompany-global-terraform-modules.git//util/data"
  environment = terraform.workspace
  vpc_name    = "${terraform.workspace}.consume"
}

```
需要定义yaml，指定vpc的值
```
ew1.production:
  environment: production
  region:      eu-west-1
  profile:     ew1.production
  namespace:   consume  # 这里就是vpc 的名字
......

```
# 调用 data的值
用法为： module.<module_name>.attribute

```
module "ec2_instance" {
  source = "terraform-aws-modules/ec2-instance/aws"
 ......
  subnet_id                   = module.data.aws_private_subnets_ids[0]  # data的动态查询。由于 aws_private_subnet是for_each创建的，必然是一个list,那么[0]，是其第一个元素。
......
  }
```
