- dynamic 是 Terraform 批量生成同名配置块的语法，核心规则：
```
• dynamic "块名"：声明要动态生成的配置块类型（如 dynamic "tag" 生成 tag 块）；
• for_each：遍历的数据源（必须是列表 / Map/Set，决定生成多少个块）；
• content { ... }：每个动态块的具体内容，用 块名.value 访问遍历的元素；
• 最终效果：等价于手动写多个同名配置块。
```

# 
```
staging:
  autoscalings:
    whatever: 
      ingress_rules:
        ELK_1:
          description: Internal Network
          type: ingress
          from_port: 9200
          to_port: 9200
          protocol: tcp
          cidr_blocks:
            - 10.109.128.0/18
            - 10.109.0.0/18
            - 10.16.0.0/16
            - 10.162.0.0/16
          security_group_id: 
            - sg-0efb898ffa42ffff4 # Network Load Balancer security group ID
   ......
```

创建资源如下：

```
resource "aws_security_group" "sg" {
  for_each    = ...
  vpc_id      = module.data.vpc_id

  dynamic "ingress" {   # ingress 作为block的name
    for_each = each.value.ingress_rules  # ingress的值，这里是一个Map.
    content {
      description     = ingress.value["description"]  # 调用方法为 ingress.XXX
      from_port       = ingress.value["from_port"]
      to_port         = ingress.value["to_port"]
      protocol        = ingress.value["protocol"]
      cidr_blocks     = try(ingress.value["cidr_blocks"], [])
      security_groups = try(ingress.value["security_group_id"], [])
    }
  }

}
```

等于手写2次规则：
```
  ingress {
    from_port   = ...
    to_port     = ...
    protocol    = ...
    cidr_blocks = ...
  }

  ingress {
  ......
  }
```
