
# for_each vs count
  
1. 创建重复的资源的时候用count/for_each等。count但是你调整了count后，那index都变了，tf认为都应该删除，所以要create_before_destory,所以尽量用for_each。
   
2. count一般会在module里面才会用到。技巧是 ` count = var.create_vpc ? 1 : 0 `, 如果有这个变量，则count=1,否则为0，也就是不创建该资源。

3. 用了count后，则有个index可以用，如下:
   
```   
resource "azurerm_virtual_machine" "main" {
  count                 = 3

  name                  = "${var.prefix}-vm-${count.index}"
```

# for_each 和for
  
- for_each （用于循环创建资源）仅支持集合（set）或映射（map），不支持列表（list）,是在计算 resource block {} 的 meta 用的，资源只接受for_each。而 for expression 是在处理values用。

-  for_each后的资源，支持 resource.name[each.key].XXX 这种调用。

# lookup vs try
lookup：Map 专属，仅处理「key 不存在」，语义更清晰；
try：通用容错，处理「任意值无效 / 表达式报错」，覆盖场景更广；
使用原则：
1. 单层 Map 取值 + 仅防 key 不存在 → 用lookup（语义化）；
2. 多层嵌套取值 / 非 Map 类型 / 防表达式报错 → 用try（唯一选择）
3. 替代关系：try可以完全替代lookup（try(map[key], default) = lookup(map, key, default)），但反之不行。
